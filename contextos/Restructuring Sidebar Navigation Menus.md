## Minha Visão de Organização por Módulo

### 🎯 Princípio Geral

A **Agência** precisa de **visão operacional e dados cruzados** entre clientes. O **Cliente** precisa de **clareza nos resultados e ações simples** (aprovar, comentar, pagar). São mentalidades diferentes e a hierarquia do menu deve refletir isso.

---

### 👤 Visão do Cliente (simplificar ao máximo)

| Módulo | Sub-itens na Sidebar | Lógica |
|---|---|---|
| **Visão Geral** | Dashboard · Tarefas | O cliente chega e vê um resumo de tudo. Tarefas são o que ele precisa fazer (aprovar posts, responder briefings). |
| **Tráfego Pago** | Resultados · Campanhas | "Resultados" = cards visuais com KPIs (quanto investiu, quanto gerou). "Campanhas" = a tabela detalhada para quem quer se aprofundar. Remover Kanban e Config do cliente — ele não precisa ver o operacional. |
| **Social Media** | Calendário · Aprovações | O calendário editorial é o mais importante para o cliente. "Aprovações" = fila do que precisa do OK dele, com botão de aprovar/reprovar inline. |
| **Web** | Projetos · Entregas | "Projetos" = visão de progresso (barra de %). "Entregas" = links para preview dos sites/landing pages. |
| **Financeiro** | Faturas | Visão limpa das faturas, status de pagamento, botão de pagar. Sem complexidade. |

> **Remover do cliente:** CRM e Tecnologia. O cliente não interage com o CRM da agência. Se houver algo relevante (ex: status de integração do Pixel), colocar como card dentro de "Web".

---

### 🏢 Visão da Agência (eficiência operacional)

| Módulo | Sub-itens na Sidebar | Lógica |
|---|---|---|
| **Dashboard** | *(sem sub-itens, é a home)* | Visão geral: receita, tarefas pendentes, alertas de clientes, próximos vencimentos. |
| **Clientes** | Overview · Onboarding | "Overview" = grid com todos os clientes e status resumido. "Onboarding" = pipeline de novos clientes em andamento. |
| **Tráfego** | Dashboard · Campanhas · Otimizações | Visão cross-client de performance. "Otimizações" = fila de ações sugeridas (pausar campanha X, aumentar budget Y). |
| **Social Media** | Calendário · Produção · Aprovações | "Produção" = kanban de criação de conteúdo. "Aprovações" = o que está esperando o cliente. |
| **CRM & Tech** | Integrações · Automações | Controle de APIs conectadas, pixels, webhooks. Onde a agência monitora a saúde técnica de cada cliente. |
| **Financeiro** | Faturas · Receita · Inadimplência | Visão financeira completa. "Inadimplência" como filtro rápido para cobranças. |
| **Relatórios** | Gerar · Histórico | Separar a ação de gerar do acervo de relatórios já gerados. |
| **Agente IA** | Chat · Prompts · Logs | "Prompts" = templates salvos. "Logs" = histórico de interações. |

---

### 💡 Mudanças-chave que eu faria

1. **Remover CRM do Cliente.** Ele não precisa ver isso. Qualquer info técnica relevante vai como card dentro de outro módulo.

2. **Renomear "Geral" para "Visão Geral"** na visão do cliente — é mais intuitivo.

3. **Criar "Aprovações" como sub-item dedicado** no Social Media do cliente. Hoje as aprovações estão misturadas no calendário, mas separar cria uma fila de ação clara — o cliente entra, vê o que precisa aprovar, resolve e sai.

4. **Na Agência, adicionar "Clientes" como módulo próprio** ao invés de ser só um filtro. Permite navegação por cliente e visão de saúde individual (NPS, churn risk, etc.).

5. **Tráfego do cliente: simplificar para 2 visões** (Resultados visuais + Tabela detalhada). O Kanban e Config são ferramentas internas da agência, não do cliente.

---

### Em resumo

> **Cliente** = ver resultados + aprovar coisas + pagar faturas. Máximo 4-5 módulos com 2 sub-itens cada.
>
> **Agência** = gerenciar operação + analisar dados cross-client + acompanhar saúde financeira. Módulos mais densos com 2-3 sub-itens operacionais.
